# 2026-03-24

- Gershon wants to build a custom outreach system inside OpenClaw, inspired by tools like Instantly / Smartlead / EmailBison, but better.
- Core vision: support many inboxes, outreach campaigns, sequencing, reply handling, AI personalization, scheduling, deliverability guardrails, and reporting.
- Architectural direction:
  - OpenClaw acts as the conversational brain/orchestrator.
  - A dedicated backend service ("outreach-core") handles scheduling, sending, inbox auth, queueing, retries, webhooks, and persistence.
  - Use Postgres for core data; likely Redis/worker queue for send scheduling.
- Desired entities/features:
  - inboxes, inbox health, leads, campaigns, campaign steps, messages, replies, suppression lists, templates, segments, send events.
  - lead states such as new, queued, contacted, replied, interested, meeting_booked, not_now, bounced, unsubscribed, blocked, finished.
- MVP direction:
  - inbox management
  - lead import/dedupe/segmentation
  - multi-step campaigns
  - business-hour/timezone sending
  - stop on reply
  - stats/reporting
  - reply classification
- Later phases:
  - AI personalization from websites/company context
  - smart inbox rotation and health-aware throttling
  - deliverability guardrails
  - CRM sync and richer analytics
- Suggested skill/product names discussed: outreach, mailforge, sequence-engine, clawreach.
- We should come back and build this for real.
